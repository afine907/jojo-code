# Agent 4: 代码质量审查

## 审查范围

审查所有代码文件的质量、风格和可维护性。

## 审查标准

### Python 代码 (使用 ruff 检查)

```bash
# 检查代码风格
ruff check src/ --output-format=json

# 检查复杂度
ruff check src/ --select=C90

# 检查文档
ruff check src/ --select=D
```

### TypeScript 代码

检查项:
- [ ] 类型安全 (no-any)
- [ ] 未使用变量
- [ ] 空值检查
- [ ] 异步处理

## 详细审查

### 1. 命名规范

检查:
- [ ] 变量命名是否清晰
- [ ] 函数命名是否表达意图
- [ ] 常量是否使用大写
- [ ] 私有方法是否使用下划线前缀

### 2. 函数复杂度

检查每个函数:
- [ ] 圈复杂度 < 10
- [ ] 函数长度 < 50 行
- [ ] 参数数量 < 5

### 3. 代码重复

检查:
- [ ] 相似代码片段
- [ ] 重复的业务逻辑
- [ ] 可以抽取的公共函数

### 4. 注释质量

检查:
- [ ] 函数文档字符串
- [ ] 复杂逻辑解释
- [ ] TODO 标记
- [ ] 废弃代码注释

### 5. 错误处理

检查:
- [ ] 异常捕获是否具体
- [ ] 错误信息是否清晰
- [ ] 资源释放是否正确
