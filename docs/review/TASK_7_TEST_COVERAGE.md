# Agent 7: 测试覆盖率审查

## 审查范围

审查所有测试文件，评估测试覆盖率和测试质量。

## 当前测试统计

- Python: 508 tests
- TypeScript: 31 tests

## 审查要点

### 1. 测试覆盖率分析

运行覆盖率报告:
```bash
# Python
pytest --cov=src --cov-report=html

# TypeScript
vitest run --coverage
```

检查项:
- [ ] 总体覆盖率 > 80%
- [ ] 核心模块覆盖率 > 90%
- [ ] 边界情况覆盖
- [ ] 错误路径覆盖

### 2. 测试质量审查

审查测试文件:
- `tests/test_server_rpc.py`
- `packages/cli/tests/*.test.ts`

检查项:
- [ ] 测试命名是否清晰
- [ ] 测试是否独立
- [ ] 是否有断言
- [ ] 是否有清理

### 3. 测试分类

统计:
- [ ] 单元测试数量
- [ ] 集成测试数量
- [ ] E2E测试数量
- [ ] 性能测试数量

### 4. 缺失测试

检查哪些模块缺少测试:
- [ ] 新增的 server 模块
- [ ] React 组件
- [ ] Hook
- [ ] 工具函数

### 5. 测试最佳实践

检查项:
- [ ] 是否使用 Mock
- [ ] 是否测试异步代码
- [ ] 是否测试错误情况
- [ ] 是否有测试数据工厂
