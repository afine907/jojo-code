# Agent 5: 错误处理审查

## 审查范围

审查所有错误处理和异常处理代码。

## 审查清单

### 1. 异常捕获

审查所有 try-except 块:

```bash
# 查找所有异常捕获
grep -r "except" src/ --include="*.py"
grep -r "catch" packages/cli/src --include="*.ts"
```

检查项:
- [ ] 是否捕获具体异常
- [ ] 是否有裸 except
- [ ] 是否正确重新抛出
- [ ] 是否记录错误日志

### 2. 边界条件

审查文件:
- 所有用户输入处理
- 所有文件操作
- 所有网络请求

检查项:
- [ ] 空值处理
- [ ] 空数组处理
- [ ] 超大输入处理
- [ ] 负数处理
- [ ] 零除处理

### 3. 资源管理

检查项:
- [ ] 文件是否正确关闭
- [ ] 连接是否正确释放
- [ ] 进程是否正确终止
- [ ] 内存是否正确释放

### 4. 超时处理

审查文件:
- `src/nano_code/tools/shell_tools.py`
- `packages/cli/src/client/rpc.ts`

检查项:
- [ ] 是否有超时设置
- [ ] 超时后是否正确清理
- [ ] 超时时间是否合理

### 5. 重试机制

检查项:
- [ ] 是否有重试逻辑
- [ ] 重试次数是否合理
- [ ] 是否有退避策略
- [ ] 是否记录重试日志

### 6. 错误消息

检查所有错误消息:
- [ ] 是否清晰描述问题
- [ ] 是否提供解决建议
- [ ] 是否避免技术术语
- [ ] 是否国际化

## 测试场景

请编写测试验证以下场景:

1. 网络超时
2. 文件不存在
3. 权限不足
4. 内存不足
5. 磁盘满
